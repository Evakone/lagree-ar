For local development with camera access, you need HTTPS. Here are your options:

‚ö†Ô∏è Camera Access Issue: Modern browsers require HTTPS for camera access. Local HTTP servers won't work for AR testing.

Option 1: Use the HTTPS Server Script

Run this in your terminal:

./start-local-server.sh

Then open: https://localhost:8000

Option 2: Use ngrok (Recommended)

Install ngrok and run:

ngrok http 8000

Use the HTTPS URL provided by ngrok

Option 3: Use Vercel Dev

Run this in your terminal:

vercel dev

This will give you a local HTTPS server

üéØ Debug Tool Usage

The debug tool is the fastest way to scale your model:

  1. Print your marker using the print marker page
  2. Open the debug tool on your phone
  3. Point camera at the printed marker
  4. Adjust sliders until model fits perfectly
  5. Copy the generated code

üîç Troubleshooting

Camera Not Working?

  • Make sure you're using HTTPS (not HTTP)
  • Check browser permissions for camera access
  • Try on a different device/browser

Marker Not Loading?

  • Check if /assets/markers/targets.mind exists
  • Verify the file is not corrupted
  • Try regenerating the marker file

Model Not Scaling?

  • Use the debug tool for real-time adjustment
  • Check the console for errors
  • Verify the model file is loading correctly
